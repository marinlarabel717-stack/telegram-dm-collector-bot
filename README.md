# telegram-dm-collector-bot

一个用于 **收集 Telegram 私信用户资料**，并支持 **上传 session 账号后采集频道帖子中的 @用户名 / 群组历史发言用户** 的机器人项目。

当前版本：`0.2.30`

## 第一版已完成

- 基于 **python-telegram-bot 20+**
- 管理端全部使用 **内联按钮交互**
- 正文消息统一走：`<tg-emoji ...>` + `parse_mode='HTML'`
- 按钮图标统一走：`api_kwargs['icon_custom_emoji_id']`
- 支持上传 `.session` 文件，或上传包含 `.session + .json` 的 `.zip` 并立即验证账号状态
- 管理员直接发送 `.session/.zip` 也会自动处理，不再依赖先点按钮保留状态
- session 验证阶段如果遇到兼容异常，也会先入库并标记为“异常”，方便后续继续排查
- 对新版/扩展 schema 的 Telethon session 增加了兼容转换层，会自动生成兼容副本再尝试验证
- 去重导出时会自动排除用户名以 `bot` 结尾的机器人账号
- 按钮只保留 1 个会员 emoji 图标，不再在文字里叠加普通 emoji
- 导航类按钮改为真正按语义使用会员 emoji：返回=`⬅️`、下一页=`➡️`、首页=`🏠`
- 频道导入支持文本消息和 txt 文件两种方式
- 结果导出除用户名外，还会附带失败/跳过频道及原因
- 机器人重启后会自动回收未完成采集任务，避免账号长期卡在“采集中”
- 点击“停止任务”会立即停止任务并释放对应采集账号
- 遇到损坏的 `.session`（如 `database disk image is malformed`）时，会只标记当前账号异常并跳过，不再让整条采集任务直接崩掉
- 任务状态中的 `running` 现显示为“采集中”，不再误显示成“异常”
- 任务详情里的频道子任务默认隐藏“已完成”，优先展示进行中 / 排队中 / 异常 / 已停止
- 自动刷新进度频率已下调，避免高频刷消息
- 账号详情会直接显示中文白话结果，如“session 已损坏 / 已失效或已封禁”
- 账号管理新增“批量检测”，会把坏 session、封禁/失效账号单独归类
- 检测到损坏 / 封禁 / 失效账号后会自动删除，对外账号列表只保留存活账号
- 账号管理页会直接显示可用 / 检测中 / 采集中数量，以及待清理无效数量
- 账号管理页和账号列表页都提供更明显的“一键检测全部 / 一键清理无效”入口
- 点击“一键检测全部”时会实时显示批量检测进度，并改为最多 10 路并发检测，避免账号多时误以为卡住
- 采集线程选择会按“已选账号数 / 频道数 / 并发上限”自动限制，不再出现能选但实际跑不到的线程数
- 采集并发上限默认提升到 50，并支持更高档位按钮 / 自定义线程数输入
- 任务详情页等刷新类按钮已改用专用 refresh emoji，并避免同一页重复堆相同图标
- 任务/历史列表中的“已完成”状态徽标已改成专用绿色 `🟢` custom emoji，避免和“已停止”等状态混淆
- 任务列表现已支持每页 6 条分页展示（更适合两列按钮排版）；列表页只保留“查看任务”，删除统一放到任务详情页
- 历史结果页也改为点击任务进入详情后再删除；任务列表页新增“一键清空任务历史”
- 上传 `.session` / `.zip` / 频道 `.txt` 后，会先立即回复“正在处理”，避免用户误以为卡住
- 账号显示编号改为面向用户的连续编号；即使前面的账号被删掉，剩余账号也会自动顺延，不再跳号
- 任务显示编号也改为连续编号；清空任务历史后，新任务会从 `#1` 重新开始显示，不再沿用旧的数据库自增 id
- 账号列表 / 账号选择等顺序统一改为按编号从大到小排列，保证像 `#10` 这类最新编号会出现在第一页最前面
- “采集中心”已改名为“采集用户”；点击“新建采集任务”后可直接选择“采集频道 / 采集群组”
- 已接入第一版 **群组发言用户采集（隐藏成员）**：支持公开群 / 私密邀请链接 / joinchat 链接，账号会先尝试入群，再读取群历史消息提取发言用户
- 群组发言采集支持可选过滤：机器人 / 管理员 / 无头像 / 无用户名
- 群组发言采集支持导出 3 份文件：去重 `@用户名`、无用户名用户 `ID`、失败群组及原因
- 单账号每轮最多新增 5 个群；剩余群组会自动等待冷却后继续处理，直到全部跑完
- 主菜单、账号管理、采集中心、任务/历史等菜单按钮统一优化为每行 2 个，更整齐直观
- 上传或批量检测后，如果没有保留下任何可用账号，会直接给出白话提示
- 账号列表可查看：
  - 可用
  - 检测中
  - 未登录
  - 异常
  - 采集中
- 支持新建采集任务：
  - 多频道
  - 可直接发送频道列表
  - 可上传 `.txt` 频道列表
  - 选择最近几天消息
  - 多账号并发
  - 提取帖子正文中的 `@username`
  - 自动去重
  - 不存在/失败频道自动跳过，不阻断整批任务
  - 生成 txt 结果文件（附失败频道原因）
- 支持查看任务进度、停止任务、导出结果
- 保留原先 DM 收集链路（用户资料 / 私信入库 / 可选转发管理员）

## 配置

复制 `.env.example` 为 `.env`，至少填写：

```env
BOT_TOKEN=你的机器人Token
ADMIN_IDS=你的Telegram数字ID
```

### 关键环境变量

- `BOT_TOKEN`：机器人 token
- `ADMIN_IDS`：管理员 ID，多个用英文逗号分隔
- 用户 session 连接已固定使用 Telegram 官方 `2040 / b18441a1ff607e10a989891a5462e627`
- `DATA_DIR`：数据目录
- `DB_PATH`：SQLite 路径
- `SESSION_DIR`：session 文件目录
- `EXPORT_DIR`：导出结果目录
- `MAX_COLLECT_WORKERS`：采集最大并发 worker 数，默认 50，实际上会按已选账号数 / 频道数 / 上限自动取最小值

### 会员 emoji 相关

支持自定义这些 custom emoji id：

- `EMOJI_WELCOME_ID`
- `EMOJI_INBOX_ID`
- `EMOJI_STATS_ID`
- `EMOJI_EXPORT_ID`
- `EMOJI_SUCCESS_ID`
- `EMOJI_UPLOAD_ID`
- `EMOJI_WAITING_ID`
- `EMOJI_OK_ID`
- `EMOJI_ERROR_ID`

## 第一版交互说明

### 1. 上传账号
- 进入 **账号管理**
- 点击 **上传 session**
- 发送 `.session` 文件，或发送包含 `.session + .json` 的 `.zip`
- 机器人会自动保存/解压并验证账号状态

### 2. 新建采集
- 进入 **采集用户**
- 点击 **新建采集任务**
- 选择 **采集频道** 或 **采集群组**
- 频道采集：发送频道列表（一行一个），或直接上传 `.txt` 频道文件
- 群组采集：发送群链接 / 群用户名（一行一个），或直接上传 `.txt` 群组文件
- 选择天数（支持自定义）
- 群组采集可继续设置过滤规则
- 选择账号
- 选择并发
- 确认启动

### 3. 查看结果
- 在任务详情页点击 **导出结果**
- 或在 **历史结果** 里重新导出

导出的 txt 内容格式：

```text
# 去重用户名结果

@username1
@username2
@username3

# 失败/跳过频道

@not_exists_channel | No user has "not_exists_channel" as username
@private_channel | Channel private
```

## 当前限制（第一版已知）

- 目前只支持上传已登录的 `.session` 文件，不包含手机号验证码登录流程
- 频道/群组采集都依赖上传账号本身是否有权限访问目标对象
- 群组发言采集当前基于历史消息里的真实发言者，不做完整成员表遍历
- 提取范围目前以消息正文中的 `@用户名` 或群历史发言用户为主，不含按钮链接深挖
- 任务停止后会保留已采集到的部分结果，可继续导出
